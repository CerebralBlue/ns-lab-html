    Module 1.3 - Integrate Virtual Agent with NeuralSeek

    Integrate

    Navigate to the ‚ÄúIntegrate‚Äù tab in NeuralSeek.

    Select Virtual Agent

    Select the preferred choice of virtual agent on the left side menu. For this lab, we recommend "LexV2 Lambda".

    Download File

    Download the Lambda Archive .zip file to your local storage.

    Create a Function

    Click the link to open the Functions page on the AWS Lambda console to create a function from scratch. - (a) Select "Create Function". - (b) Add a ‚ÄúFunction Name‚Äù. For this lab, we recommend ‚Äúlearning-lab‚Äù. - (c) Click ‚ÄúCreate Function‚Äù.

    Upload the downloaded .zip File

    Navigate to the Code Source pane in the AWS Lambda Function console. - (a) Click ‚ÄúUpload from‚Äù. Click ‚Äú.zip file‚Äù. - (b) Click ‚ÄúUpload‚Äù to select your Lambda Archive .zip file you downloaded. - (c) Click ‚ÄúSave‚Äù.

    Add API Key and Instance URL

    On the NeuralSeek "Integrate" page, copy the provided API key and Instance URL mentioned in the NeuralSeek's integration page. Navigate to the Code Source pane in the AWS Lambda Function console. - (a) Click on the ‚Äúindex.mjs‚Äù file tab. - (b) Enter the copied API key and Instance URL into the code block:

    const neuralSeekURL = ‚Äúenter url here‚Äù ;

    {% endraw %}

    • The "Deploy" button will enable. Click "Deploy" to successfully update and deploy the function.

    Update Timeout Setting

    Navigate to the "Configuration" tab in AWS Lambda Function Console.

    • (a) In the "General Configuration" pane, click "Edit".
    • (b) Update under ‚ÄúTimeout‚Äù: set min to be ‚Äú1‚Äù and sec to be ‚Äú0‚Äù. This will ensure the lambda function will not time out for 1 minute.
    • (c) Click "Save".

    Create a Lex Bot

    1. Open the Amazon Lex console. click "Create bot" under the Bots section.
    2. In the creation method, select "Create a blank bot".
    3. Name the bot "TestBot".
    4. Select "Create a role with basic Amazon Lex permissions" in IAM Permissions.
    5. Select "No" in Children's Online Privacy Protection Act (COPPA).
    6. Click "Next".
    7. Click "Done".
    8. In the Sample utterances section, add an utterance. It can be anything (e.g. Hello).
    9. In the Initial response section, add any response that may be suitable. (e.g. Welcome to the test bot).
    10. Click "Save intent" to save a default intent without adding anything.

    Selecting Alias

    Open the Amazon Lex console. Select the "TestBot" from the list of the Bots. - (a) On the left side bar menu, under ‚ÄúDeployment‚Äù, click ‚ÄúAliases‚Äù. - (b) From the list of alias names, choose the alias name that you want to use. Select "TestBotAlias".

    Select Language

    Under the chosen Alias, "TestBotAlias" within the Amazon Lex console:

    • (a) From the list of supported languages, click the language that the Lambda function is used for. For this lab, we recommend ‚ÄúEnglish‚Äù.

    Optional Lambda Function Connection

    • (a) Under "Source", click the name of the alias of the Lambda function to use. For this lab, we recommend "learning-lab‚Äù.
    • (b) Under "Lambda function version or alias", the option "$LATEST" will populate.
    • (c) Click ‚ÄúSave‚Äù.

    Fallback Activities

    The default FallbackIntent allows for the Lambda function to be called when it is detected. The idea is that Lex would invoke NeuralSeek to answer whenever there is no matching intent found. To activate this, navigate to Amazon Lex console.

    • (a) Click ‚ÄúTestBot‚Äù
    • (b) Click ‚ÄúIntents‚Äù under ‚ÄúEnglish (US)‚Äù on the left sidebar menu.
    • (c) Click on the ‚ÄúFallbackIntent‚Äù link.

    Under the "Fulfillment" section, click the "Activate" icon to turn on.

    • (a) Click "Advanced Options"
    • (b) Select the ‚ÄúUse a Lambda Function for Fulfillment‚Äù option.
    • (c) Click "Update Options."
    • Click "Save Intent."

    Build the chatbot and test the FallbackIntent routine with a question.

    • (a) Click ‚ÄúBuild.‚Äù This step may take a few moments.
    • (b) Click ‚ÄúTest.‚Äù The Amazon Lex chatbot will be generated.
    • Prompt the chatbot with a question. For the purpose of this lab, we used ‚ÄúWhat is NeuralSeek?‚Äù
    • Notice the answer and the detailed information about the answer is returned to the chat bot.

    The AWS LexV2 Lambda Virtual Agent is now set up, integrated with NeuralSeek, tested and ready for use.
