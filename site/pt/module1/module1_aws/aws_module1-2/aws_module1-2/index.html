    1.2 - Configurar o NeuralSeek

    Conceitos básicos - Começando

    Primeiro, precisamos de um breve texto sobre sua empresa ou organização, enfatizando o caso de uso para esta instância do NeuralSeek.

    • (a) Adicione uma breve descrição da empresa ou organização.
    • (b) Selecione o Idioma de Saída no menu suspenso. Neste exemplo, selecione Inglês. É melhor combinar o idioma da sua Base de Conhecimento e perguntas, mas o NeuralSeek é capaz de traduzir entre muitos idiomas.
    • (c) Selecione o uso preferido da instância do NeuralSeek. Esta instância do NeuralSeek será usada internamente em sua organização ou será externa - voltada para o cliente? Usaremos essas informações para pré-ajustar alguns parâmetros de segurança para você. Neste exemplo, selecione Interno.

    Clique em Avançar para salvar uma descrição única, o idioma e a escolha de caso de uso.

    Dados - Conectar-se à sua Base de Conhecimento

    • (a) Selecione a Base de Conhecimento desejada no menu suspenso. Para fins deste laboratório, recomendamos a Base de Conhecimento da AWS Kendra.
    • (b) Selecione o idioma desejado. Para fins deste laboratório, clique em Inglês.

    Adicionar informações de Base de Conhecimento

    O usuário é responsável por fornecer informações detalhadas da Base de Conhecimento selecionada. Como leva tempo para configurar essas informações para si mesmo, usaremos uma que já está preparada para o aprendizado do laboratório, com acesso somente leitura. Os valores reais abaixo serão fornecidos durante o curso deste laboratório pelo instrutor do laboratório.

    Para entender como criar e localizar as chaves de acesso da AWS, consulte a seção Informações adicionais - Chaves de acesso do guia do laboratório.

    Por favor, informe o seguinte:

    • (a) Adicione o ID do índice Kendra.
    • (b) Adicione a Chave de acesso da Função da AWS.
    • (c) Adicione a Região da AWS.
    • (d) Adicione a Chave de acesso da Função Secreta da AWS.

    Clique no botão "Testar" para testar a conexão. Em seguida, clique em Avançar para salvar a configuração da Base de Conhecimento.

    O NeuralSeek agora está configurado com uma Base de Conhecimento, neste caso: Índice Kendra, e pronto para buscar.

    ⚠️ Para criar seu próprio índice Kendra para fazer o laboratório

    Com o interesse de economizar tempo, o laboratório usará um índice Kendra já integrado que está pré-preenchido com os dados. Isso ocorre devido ao fato de que a criação do índice Kendra e o preenchimento dos dados levariam de 40 minutos a mais de uma hora para serem concluídos. No entanto, se você estiver interessado em posteriormente realizar o laboratório usando seu próprio índice e chaves do Kendra, poderá seguir as instruções abaixo.

    🔗Configurando o índice Kendra e as chaves de acesso da AWS

    Organizar - Saídas e Categorias

    Categorização

    Categorize suas intenções para relatórios e gerenciamento mais fáceis.

    Se você está seguindo o laboratório ao vivo, pule esta etapa! Vamos revisitar isso no Módulo 2.

    • (a) Clique no ícone de lâmpada para adicionar uma nova linha.
    • (b) Digite um nome de categoria exclusivo, um URL correspondente e uma descrição detalhada da categoria com intenções que não correspondam a nenhuma outra categoria.

    Framework do Agente Virtual

    • (a) Selecione o Tipo de Agente Virtual no menu suspenso. Neste exemplo, selecione AWS Lex V2.
    • (b) Selecione a opção preferida de habilitação ou desabilitação de incorporação de links nas respostas retornadas. Neste exemplo, selecione Desabilitar.

    Clique em Avançar para salvar categorias relatadas e o framework selecionado do Agente Virtual.

    Ajustar - Ajustar para seus Dados

    Vamos ajustar o NeuralSeek para seus dados.

    • (a) Selecione a opção preferida para Qual destes parece mais com seus dados? Para este exemplo, selecione Frases importantes são cercadas por muitas linhas de texto que as explicam..
      • A opção Frases importantes são cercadas por muitas linhas de texto que as explicam, descreve dados com explicações mais elaboradas com passagens de texto mais extensas para uma melhor compreensão.
      • A opção Frases importantes são cercadas por apenas uma ou duas linhas de informações relevantes, descreve dados com cobertura sucinta de frases-chave com apenas uma ou duas linhas de contexto necessário.
    • (b) Selecione a opção preferida que melhor descreve suas necessidades de dados. Para este exemplo, selecione Quero fornecer muitas informações para a LLM e dar margem para que ela decida a resposta.
    • (c) Selecione a opção preferida que melhor descreve a atualidade dos seus dados. Para este exemplo, selecione Todos os meus documentos estão atualizados e relevantes.
      • A opção Todos os meus documentos estão atualizados e relevantes descreve dados que estão atualizados.
      • A opção Tenho muitos documentos antigos ou posts de blog, alguns com informações conflitantes. Preciso que o NeuralSeek me ajude com a priorização de datas descreve dados que podem conter detalhes conflitantes ou irrelevantes em diferentes fontes ou documentos.

    Clique em Próximo para salvar as configurações de ajuste relatadas.

    Perguntas e Respostas

    À medida que as perguntas são respondidas pelo NeuralSeek, podemos automaticamente construir e manter porções do seu agente virtual.

    Use os botões abaixo para gerar rapidamente conteúdo de Perguntas e Respostas (Q&A) para iniciar e testar seu agente virtual.

    • Gerar automaticamente perguntas fará uma consulta à base de dados, pedirá ao NeuralSeek para gerar insights e produzirá diversos testes de perguntas com base no conteúdo corporativo. Clique em Enviar para ter o NeuralSeek gerando respostas.

    • Inserir perguntas manualmente fornecerá uma caixa de teste em branco para que você possa inserir perguntas, uma por linha, com base no conteúdo corporativo. Clique em Enviar para que o NeuralSeek gere respostas.

    • Carregar perguntas de teste fornecerá um link para o seu navegador de arquivos local para carregar um arquivo CSV de perguntas de teste. As perguntas serão executadas no endpoint do Seek em paralelo e pontuadas. Use o modelo de arquivo .csv fornecido para criar suas perguntas. Os arquivos de entrada devem manter esses títulos de coluna como mínimo, mas você pode adicionar colunas de payload adicionais. Clique em Enviar para que o NeuralSeek gere respostas.
